# Documentation Standards

## Bilingual Documentation (EN/PL)

- **ALWAYS** create documentation in both English and Polish
- English version: `filename.md` (e.g., `README.md`, `docs/usage.md`)
- Polish version: `filename.pl.md` (e.g., `README.pl.md`, `docs/usage.pl.md`)
- Both versions must contain the same information and structure
- When creating new documentation files, create both EN and PL versions simultaneously

## Documentation Files That Must Be Bilingual

- All files in `docs/` directory
- `README.md` / `README.pl.md`
- `FAQ.md` / `FAQ.pl.md`
- `SECURITY.md` / `SECURITY.pl.md`
- `CHANGELOG.md` / `CHANGELOG.pl.md`
- `VERSION_STATUS.md` / `VERSION_STATUS.pl.md`
- Any other documentation files in the root directory

## File Naming Convention

- English: `filename.md`
- Polish: `filename.pl.md`
- Examples:
  - `docs/usage.md` → `docs/usage.pl.md`
  - `docs/HOW_IT_WORKS.md` → `docs/HOW_IT_WORKS.pl.md`
  - `FAQ.md` → `FAQ.pl.md`

## Code Comments

- Code comments can be in English only (standard practice)
- PHPDoc blocks should be in English

## Commit Messages

- Commit messages should be in English
- Polish can be used in commit body if needed for clarity
